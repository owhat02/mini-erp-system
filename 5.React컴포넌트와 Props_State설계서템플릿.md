## 1. 컴포넌트 개요

### 1.1 컴포넌트명
**ProjectDetailPage** (프로젝트 상세 및 칸반 보드 페이지)

### 1.2 목적 및 역할
- **주요 목적**: 특정 프로젝트의 상세 정보를 확인하고, 하위 업무(Task)들을 TODO, DOING, DONE 상태별로 분류하여 칸반 보드 형태로 조회 및 관리합니다.
- **위치**: `src/pages/Project/ProjectDetailPage.jsx`
- **컴포넌트 유형**: 페이지(Page) 컴포넌트

### 1.3 기능 요구사항
- [x] 프로젝트 기본 정보(제목, 기간, 전체 진척도) 표시
- [x] 업무(Task) 상태별(TODO, DOING, DONE) 3단 컬럼 렌더링
- [x] PM 권한일 경우 신규 업무 생성 버튼 및 모달 제공
- [x] 업무 카드 클릭 시 상세 정보 및 상태 변경 기능 제공

## 3. Props 설계

### 3.1 Props 인터페이스 (주요 하위 컴포넌트 기준)

**1) KanbanBoard 컴포넌트**
| Props명 | 타입 | 필수여부 | 기본값 | 설명 |
|---------|------|----------|--------|------|
| `tasks` | `Array` | 필수 | — | 프로젝트에 속한 전체 업무 목록 |
| `onStatusChange` | `Function` | 필수 | — | 업무 상태 변경 콜백 함수 (taskId, newStatus) |

**2) TaskCard 컴포넌트**
| Props명 | 타입 | 필수여부 | 기본값 | 설명 |
|---------|------|----------|--------|------|
| `task` | `Object` | 필수 | — | 표시할 단일 업무 데이터 객체 |
| `onClick` | `Function` | 필수 | — | 카드 클릭 시 상세 조회 모달을 띄우는 콜백 |

> **데이터 구조 참고 (백엔드 Entity v2.1 기반)**
> Task 객체: `{ id, taskTitle, taskContent, taskState, startDate, endDate, assignees: [{ id, name }] }`

## 4. 상태 관리 설계

### 4.1 상태 위치 결정 근거
| 상태명 | 위치 선택 | 결정 근거 |
|--------|-----------|-----------|
| `isCreateModalOpen` | 로컬 `useState` | 페이지 내에서 모달을 열고 닫는 단순 UI 상태이므로 로컬 관리 |
| `project` & `tasks` | `projectStore` | 서버에서 가져오는 핵심 도메인 데이터이며, 다른 컴포넌트에서도 참조 가능성 있음 |
| `user` | `authStore` | 현재 접속한 사용자의 정보 및 권한(PM 여부)을 판별하기 위해 전역 인증 스토어 사용 |

### 4.2 로컬 상태 (useState)
| 상태명 | 초기값 | 용도 |
|--------|--------|------|
| `isCreateModalOpen` | `false` | 신규 업무 생성 모달 표시 여부 제어 |
| `selectedTaskId` | `null` | 상세 조회/수정할 업무 카드 ID 임시 저장 |

### 4.3 전역 상태 (Zustand)
| Store명 | 상태/액션명 | 용도 |
|---------|-------------|------|
| `projectStore` | `currentProject` | 화면 상단 헤더에 표시할 프로젝트 기본 정보 |
| `projectStore` | `tasks` | 칸반 보드에 렌더링할 하위 업무 목록 배열 |
| `projectStore` | `fetchProjectDetail(id)` | 페이지 진입 시 프로젝트 및 업무 데이터를 서버에서 조회 |
| `projectStore` | `updateTaskStatus()` | 업무 카드 상태(TODO➔DOING) 변경을 서버에 요청 |

## 5. API 연동 설계

### 5.1 사용하는 API 목록
| API 엔드포인트 | 메서드 | 용도 | 호출 시점 | 인증 필요 |
|----------------|--------|------|-----------|-----------|
| `/api/v1/projects/{id}` | `GET` | 프로젝트 기본 정보 및 Task 목록 조회 | 페이지 마운트 시 (`useEffect`) | 필요 |
| `/api/v1/tasks?projectId={id}` | `GET` | 해당 프로젝트의 하위 Task 목록 조회 | 페이지 마운트 시 (`useEffect`) | 필요 |
| `/api/v1/tasks` | `POST` | 신규 업무 생성 | 모달에서 '생성' 버튼 클릭 시 | 필요 |
| `/api/v1/tasks/{id}/status` | `PATCH` | 업무 상태(TODO/DOING/DONE) 변경 | 업무 상태 드롭다운 변경 시 | 필요 |

## 6. 컴포넌트 구현 골격 (ProjectDetailPage.jsx)

```jsx
import { useState, useEffect } from 'react';
import { useParams } from 'react-router-dom';
import { useProjectStore } from '@/store/projectStore';
import { useAuthStore } from '@/store/authStore';
import ProjectHeader from './components/ProjectHeader';
import KanbanBoard from './components/KanbanBoard';
import TaskCreateModal from './components/TaskCreateModal';

export default function ProjectDetailPage() {
  // 1. 라우터 파라미터 및 로컬 상태
  const { projectId } = useParams();
  const [isCreateModalOpen, setIsCreateModalOpen] = useState(false);

  // 2. Zustand 전역 상태
  const { currentProject, tasks, loading, fetchProjectData, updateTaskStatus } = useProjectStore();
  const { user } = useAuthStore();

  // 3. 데이터 로드 (초기 렌더링 시 프로젝트 상세 및 업무 목록 동시 조회)
  useEffect(() => {
    if (projectId) {
      fetchProjectData(projectId);
    }
  }, [projectId, fetchProjectData]);

  // 4. 이벤트 핸들러
  const handleStatusChange = async (taskId, newStatus) => {
    await updateTaskStatus(taskId, newStatus);
  };

  // 5. 조건부 렌더링 (로딩 처리)
  if (loading || !currentProject) {
    return <div className="loading-spinner">프로젝트 정보를 불러오는 중...</div>;
  }

  // 6. 메인 렌더링
  return (
    <div className="project-detail-container p-6">
      <ProjectHeader 
        project={currentProject} 
        onNewTaskClick={() => setIsCreateModalOpen(true)} 
        isPM={user?.projectRole === 'PM'} 
      />

      <KanbanBoard 
        tasks={tasks} 
        onStatusChange={handleStatusChange} 
      />

      <TaskCreateModal 
        isOpen={isCreateModalOpen} 
        onClose={() => setIsCreateModalOpen(false)} 
        projectId={currentProject.id}
        members={currentProject.members} // 다중 담당자 지정을 위한 멤버 리스트 전달
      />
    </div>
  );
}
```
