## 1) 오늘 주요 이슈 요약

### 1-1. 인증 후 API에서 `USER_NOT_FOUND` 발생
- 증상
  - `GET /api/v1/users` 호출 시 사용자 조회 실패
  - `GET /api/v1/leave/balance` 호출 시 사용자 조회 실패
  - `GET /api/v1/calendar/events` 호출 시 사용자 조회 실패
  - 특근 API 일부에서도 동일 패턴 재현

- 핵심 원인
  - JWT `sub`(subject)는 `userId`(숫자)로 발급됨
  - 일부 서비스가 `authentication.getName()`을 `loginId`로 해석해 `findByLoginId(...)` 조회
  - 결과적으로 `findByLoginId("3")` 같은 잘못된 조회가 발생

- 조치 방향
  - 인증 이후 내부 식별자를 `userId` 기준으로 통일
  - `authentication.getName()` -> `Long userId`로 파싱
  - 서비스 조회를 `findById(userId)` 중심으로 변경

### 1-2. Swagger/Postman에서 401/404 혼선
- 401 원인
  - 보호 API에 `Authorization: Bearer {token}` 누락/형식 오류
  - 만료된 토큰 사용
- 404/NoResource 혼선 원인
  - 잘못된 경로 입력(점/슬래시/오타)
  - 실행 중 서버 코드 버전과 로컬 코드 버전 불일치

## 3) DB/환경 이슈 정리

### 3-1. `lab` 계정 접속 이슈
- 내 PC: `lab/lab` 접속 정상
- 동료 PC: 권한 누락 가능성 높음

권한 복구 SQL(관리자 계정에서 실행):

```sql
CREATE DATABASE IF NOT EXISTS mini_erp CHARACTER SET utf8mb4 COLLATE utf8mb4_general_ci;

CREATE USER IF NOT EXISTS 'lab'@'localhost' IDENTIFIED BY 'lab';
CREATE USER IF NOT EXISTS 'lab'@'%' IDENTIFIED BY 'lab';

ALTER USER 'lab'@'localhost' IDENTIFIED BY 'lab';
ALTER USER 'lab'@'%' IDENTIFIED BY 'lab';

GRANT ALL PRIVILEGES ON mini_erp.* TO 'lab'@'localhost';
GRANT ALL PRIVILEGES ON mini_erp.* TO 'lab'@'%';

FLUSH PRIVILEGES;
```

확인:

```sql
SHOW GRANTS FOR 'lab'@'localhost';
SHOW GRANTS FOR 'lab'@'%';
```
